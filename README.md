# OpenRouter Docs Pipeline Demo

Takes an OpenAPI spec, enriches it with code snippets via Speakeasy, and outputs Mintlify-ready MDX files.

```
OpenAPI spec → Speakeasy SDKs → snippets.json → enriched spec → MDX files → mint.json
```

## Prerequisites

| Tool | Install |
|------|---------|
| Python 3.9+ | `brew install python` |
| PyYAML | installed automatically by `run-demo.sh` |
| Speakeasy CLI | `brew install speakeasy-api/tap/speakeasy` |
| Mintlify (preview only) | `npm i -g mintlify` or use `npx` |

Speakeasy also requires a free API key — run `speakeasy auth` once before using the pipeline.

## Quickstart

```bash
cd demo1
./run-demo.sh
```

That's it. Output lands in `./docs-output/`. To preview:

```bash
cd docs-output
npx mintlify dev
```

## Flags

```bash
./run-demo.sh --dry-run   # skip SDK generation, use existing ./speakeasy-{python,typescript}/
```

Use `--dry-run` if you've already generated the SDKs and just want to re-run the
extraction/MDX/scaffolding stages — it's much faster and works offline.

## Can it be run multiple times?

Yes. Every run overwrites its outputs cleanly:

| Output | Overwritten each run |
|--------|----------------------|
| `./speakeasy-python/` | yes (full regen, skipped with `--dry-run`) |
| `./speakeasy-typescript/` | yes (full regen, skipped with `--dry-run`) |
| `./snippets.json` | yes |
| `./openrouter-openapi-enriched.yaml` | yes |
| `./docs-output/` | yes (all MDX, mint.json, introduction.mdx) |

The input spec (`./openrouter-openapi.yaml`) is never modified.

## Pipeline stages

| Step | Script | Input | Output |
|------|--------|-------|--------|
| 1 | `speakeasy generate sdk` | `openrouter-openapi.yaml` | `speakeasy-python/`, `speakeasy-typescript/` |
| 2 | `scripts/extract-snippets.py` | SDK dirs | `snippets.json` |
| 3 | `scripts/inject-code-samples.py` | spec + snippets | `openrouter-openapi-enriched.yaml` |
| 4 | `scripts/generate-mdx.py` | enriched spec | `docs-output/api-reference/**/*.mdx` |
| 5 | `scripts/scaffold-mint.py` | MDX folder | `docs-output/mint.json`, `docs-output/introduction.mdx` |

Each script is independently runnable. Pass `--help` for options:

```bash
python3 scripts/extract-snippets.py --help
python3 scripts/inject-code-samples.py --help
python3 scripts/generate-mdx.py --help
python3 scripts/scaffold-mint.py --help
```

## Graceful degradation

If Speakeasy SDK generation fails (no auth, network issue, etc.) the pipeline
continues without code snippets. MDX files are still generated for every operation —
they just omit the `<CodeGroup>` block. `--dry-run` with empty SDK dirs has the
same effect.

## File structure

```
demo1/
├── run-demo.sh                         # main entrypoint
├── requirements.txt                    # PyYAML
├── openrouter-openapi.yaml             # input spec (untouched)
├── openrouter-openapi-enriched.yaml    # generated — spec + x-codeSamples
├── snippets.json                       # generated — operationId → {python, typescript}
├── scripts/
│   ├── extract-snippets.py
│   ├── inject-code-samples.py
│   ├── generate-mdx.py
│   └── scaffold-mint.py
├── speakeasy-python/                   # generated by speakeasy
├── speakeasy-typescript/               # generated by speakeasy
└── docs-output/
    ├── mint.json
    ├── introduction.mdx
    └── api-reference/
        └── <tag>/
            └── <operation>.mdx
```
